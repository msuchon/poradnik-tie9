Poradnik GitHub


GitHub jest platformą, działającą na zasadzie hostingu, przeznaczoną na projekty progamistyczne. Dzięki temu serwisowi możesz za darmo podzielić się z innymi tym, nad czym aktualnie pracujesz tak, aby inni mogli z tego skorzystać bądź współtworzyć z Tobą! Portal stworzony został przez Chrisa Wanstrath‘a, PJ Hyett oraz Toma Prestona-Wernera i działa od kwietnia 2008 roku. W 2018 serwis przejęty został przez firmę Microsoft. GitHub opiera się o system kontroli wersji o nazwie Git.



Git to rozproszony system kontroli wersji, oparty o licencję GNU GPL, stworzony przez twórcę jądra Linux - Linusa Torvaldsa przy współpracy z Junio Hamano. Pierwsze wydanie systemu miało miejsce 7 kwietnia 2005 roku.

System kontroli wersji - oprogramowanie, pozwalające śledzić zmiany w kodzie źródłowym oraz łączące zmiany dokonywane na plikach w różnym czasie przez wiele osób.



Aby wygodnie pracować w GitHub, zapoznaj się z interfejsem serwisu oraz załóż konto, które można zarejestrować za darmo. Niniejszy poradnik pomoże ci zrozumieć, jak działać na GitHubie oraz w jaki sposób opublikować swój projekt! W menu powyżej, w "Spisie Treści" znajdziesz poszczególne części poradnika lub możesz przejść bezpośrednio do początku, klikając tutaj.
